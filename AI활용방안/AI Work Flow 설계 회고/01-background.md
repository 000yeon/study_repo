# Part 1 — 왜 만들었나: Admin AI 워크플로우의 배경

> "AI를 쓰는데 왜 매번 같은 설명을 반복해야 하지?"

---

## Admin 프로젝트의 현실

우리 팀은 동시에 3개의 Admin 프로젝트를 운영하고 있었다. 각 프로젝트는 달랐지만 하는 일은 거의 같았다.

- 목록 조회 → 등록 → 수정 → 삭제
- 필터 + 검색 + 페이지네이션
- 권한 분기 + 로그인 세션 관리
- Figma 디자인을 보고 UI 구현

이 반복 패턴은 **AI에게 위임하기 가장 좋은 구조**다. 규칙이 명확하고, 결과물을 검증하기 쉽고, 같은 패턴이 계속 반복된다.

문제는 **AI가 프로젝트를 모른다는 것**이었다.

---

## 기존 방식의 한계

### 팀원마다 제각각인 AI 사용 방식

Claude나 GPT를 쓰는 방식이 팀원마다 달랐다. 어떤 개발자는 Figma 스크린샷을 붙여넣고, 어떤 개발자는 기존 코드를 복사해서 붙여넣었다. 결과물의 품질 편차가 컸고, 같은 프로젝트인데 파일 구조와 패턴이 달라지는 일이 생겼다.

### 컨텍스트 재입력 비용

매 작업 세션을 시작할 때마다 AI에게 프로젝트 구조, 사용하는 라이브러리, 코딩 컨벤션을 처음부터 설명해야 했다. 작업 전 컨텍스트 세팅에만 **평균 15~20분**이 소요됐다. 하루에 세션을 3번 시작하면 1시간이 컨텍스트 세팅에 쓰인다.

### AI가 만든 코드의 재작업

컨텍스트가 부족한 상태에서 AI가 만든 코드는 프로젝트 구조와 맞지 않는 경우가 많았다. 경로 구조가 다르거나, 팀에서 쓰지 않는 라이브러리를 import하거나, 이미 만들어진 공통 컴포넌트 대신 새로 만들어버리는 일이 반복됐다. **도입 전 AI 생성 코드의 재작업률은 약 38%** 였다.

### Figma와 코드 사이의 갭

디자이너가 넘겨준 Figma 파일을 보고 개발자가 수동으로 색상 값과 간격을 코드에 입력했다. 이 과정에서 색상 토큰 오차가 평균 18% 수준으로 발생했다. 픽셀이 맞지 않고, 디자이너와 개발자 사이에 미세한 조율이 필요한 QA 라운드가 늘었다.

---

## 문제의 본질

결국 하나의 문제로 귀결됐다.

> **AI는 능력이 있는데 우리 프로젝트를 모른다.**

사람 팀원이 새로 합류하면 온보딩 문서를 주고, 코드베이스를 설명하고, 코딩 컨벤션을 공유한다. AI에게는 그런 절차 없이 바로 "구현해줘"를 던지고 있었다.

---

## 해결 방향: AI를 팀원처럼 온보딩하기

발상의 전환은 간단했다.

**AI를 도구가 아닌 팀원으로 보기.**

팀원이 프로젝트에 투입될 때 필요한 것들 — 프로젝트 개요, 디렉토리 구조, 라우팅 규칙, 디자인 시스템, 코딩 컨벤션 — 을 AI에게도 동일하게 제공하면 된다. 한 번 잘 만들어두면 모든 팀원이 모든 프로젝트에서 재사용할 수 있다.

이것이 **Admin AI Workflow**의 출발점이었다.

---

## 만들기로 결정한 이유

비슷한 도구가 없었던 것은 아니다. `cursor rules`, `.cursorrules`, GitHub Copilot의 커스텀 인스트럭션 같은 것들이 있었다. 하지만 이것들은 공통적으로 한계가 있었다.

- **단일 규칙 파일** — 프로젝트 전체 컨텍스트를 담기엔 너무 좁다
- **도구 종속** — Cursor에서 설정한 것이 Claude Code에서 동작하지 않는다
- **팀 공유 불가** — 개인 설정이라 팀 표준으로 쓸 수 없다
- **업데이트 관리 없음** — 프로젝트가 바뀌면 규칙도 바뀌어야 하는데 추적이 안 된다

우리가 원한 것은 **어떤 AI 도구에도 동작하고, 팀 전체가 쓰고, 프로젝트와 함께 버전 관리되는 컨텍스트 시스템**이었다.

---

## 목표 정의

이 시스템을 만들 때 세운 목표는 세 가지였다.

1. **AI 재작업률 20% 이하로 낮추기** — 컨텍스트가 충분하면 AI가 처음부터 맞는 코드를 만든다
2. **신규 도메인 추가 시간 절반으로 줄이기** — CRUD 하나 추가에 3일 이상 걸리지 않도록
3. **팀원 누구나 같은 방식으로 AI 사용하기** — 결과물의 품질 편차를 없애기

이 목표들이 달성됐는지는 [Part 4 — 적용 결과와 수치](./04-results-and-metrics.md)에서 확인할 수 있다.
