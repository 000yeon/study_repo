Tech Decisions
5가지 핵심 기술 선택
각 결정은 "2025년 커뮤니티 방향성과 일치하는가"를 기준으로 했다
pnpm
vs npm / yarn
content-addressable store로 디스크 60~80% 절약. strict isolation으로 phantom dependency 차단. 모노레포 네이티브 지원. 팀 3개 프로젝트 동시 운영 환경에서 효과 극대화.
Performance
Vite
vs Next.js (CSR Admin)
Admin SPA는 SEO 불필요 · SSR 불필요 · CSR 최적. Vite ESM 기반 개발 서버로 시작 시간 40초 → 1초 미만. Next.js의 SSR 기능은 Admin에서 오버헤드.
DX
ESLint v9
Flat Config
vs .eslintrc (deprecated)
v9부터 .eslintrc 공식 deprecated. Flat Config는 설정 배열이 선언 순서대로 적용되어 예측 가능. TypeScript 통합이 자연스럽고, ignores가 단일 파일에서 관리됨.
2025 Trend
Prettier
+ plugins
tailwindcss · sort-imports
ESLint와 역할 분리 (품질 vs 포맷). prettier-plugin-tailwindcss로 Tailwind 클래스 자동 정렬 — 도입 후 관련 PR 코멘트 0건. import 순서 자동화.
Code Quality
Husky v9
+ lint-staged
+ commitlint
pre-commit 게이팅
CI 전에 로컬에서 오류를 잡아 피드백 루프 단축. lint-staged로 변경 파일만 처리해 속도 유지. Conventional Commits 강제 → git log 의미 있어짐, 커밋 준수율 62% → 100%.
